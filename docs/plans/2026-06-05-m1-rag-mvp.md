# M1 RAG MVP Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Complete a working M1 knowledge RAG MVP on top of the current dataset/document metadata foundation.

**Architecture:** Keep HTTP, RBAC, storage, orchestration, and persistence in `backend`. Put pure RAG parsing/chunking/retrieval/citation/trace models in `crates/novex-rag`. Use PostgreSQL for M1 local fallback chunks and traces, while defining adapter boundaries for future Milvus, embedding, rerank, and answer model integrations.

**Tech Stack:** Rust workspace, Axum, SQLx, PostgreSQL, Next.js admin, Vitest, existing local file storage.
