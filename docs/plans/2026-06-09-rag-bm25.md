# RAG BM25 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace Novex's sparse RAG fallback ranking with local BM25 so Chinese long-document retrieval is less dominated by high-frequency character overlap.

**Architecture:** Keep the existing public `keyword_retrieve(query, chunks, limit)` API and implement BM25 behind it in `crates/novex-rag`. The retriever builds per-query in-memory corpus statistics over loaded chunks, tokenizes mixed Chinese/ASCII text into practical search terms, scores chunks with BM25 plus small exact-phrase/date boosts, and returns existing `RetrievalHit` values for backend hybrid fusion.

**Tech Stack:** Rust, `crates/novex-rag`, existing unit tests via `cargo test -p novex-rag`.

BM25 score formula:

```rust
let idf = (((document_count as f32 - df as f32 + 0.5) / (df as f32 + 0.5)) + 1.0).ln();
let denominator = tf + BM25_K1 * (1.0 - BM25_B + BM25_B * doc_len / avg_doc_len);
score += idf * (tf * (BM25_K1 + 1.0)) / denominator;
```
