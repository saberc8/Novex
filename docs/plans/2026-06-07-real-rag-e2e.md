# Real RAG E2E Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make Novex's knowledge ask path generate answers through the configured live RAG chain: embedding, Milvus recall, rerank, LLM answer, citations, and trace metadata.

**Architecture:** Keep `KnowledgeService` as the orchestrator and reuse existing `novex-model` provider calls. Add a small RAG answer-generation helper that builds grounded prompts from reranked chunks and converts `ModelChatResp` into `RagAnswer`, while preserving the existing extractive answer as an offline fallback. Add strict/live test gates so acceptance cannot pass on local fallback behavior.

**Tech Stack:** Rust, `backend-rust`, `novex-rag`, `novex-model`, SQLx/PostgreSQL, Milvus REST, `reqwest`, parser-worker Python unittest, gated live tests loaded from `backend/.env`.
